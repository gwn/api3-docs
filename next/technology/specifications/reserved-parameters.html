# Reserved parameters

Table of Contents

  • _type
  • _path
  • _times

A requester can pass request parameters either by referencing a template that contains them, or as an argument of the request-making methods of Airnode.sol. In either case, these parameters are encoded in a bytes-type variable using Airnode ABI. There are two types of parameters:

  1. Endpoint parameters (opens new window) mapped to API operation parameters
  2. Reserved parameters (opens new window)

Reserved parameters signal to the provider to perform a specific operation while fulfilling the request. Reserved parameter names start with _.

# _type

Can be int256, bool, or bytes32. Signifies what Solidity type the API response will be typecast to before fulfillment. See the conversion behavior docs (opens new window) for details.

# _path

Assuming that the API response will be a JSON object, defines the field to be used to fulfill the request using dot notation. For example, if the API returns

{
  "field1": {
    "fieldA": [
      "valueA1",
      "valueA2"
    ],
    "fieldB: "valueB"
  },
  "field2": {
    "fieldZ": "valueZ"
  }
}

If the response is a literal value (i.e., not a JSON object) and _path is not provided, Airnode will use the literal value to fulfill the request.

# _times

If _type is int256 and a _times parameter is provided, Airnode multiplies the value returned by the API with the _times parameter before fulfilling the request. For example, if the API returns:

{
  "data": "1.238",
  "apiVersion": "1.0.4"
}
